# 热图置信度方法说明（中文版，易懂版）

## 1. 这个置信度到底想解决什么问题？

我们现在的模型会先预测一张 heatmap，然后再从 heatmap 里解出 2D 点的位置。
问题在于：不是每个视角、每个点的预测都一样可靠。

比如：

- 点的位置很清楚，heatmap 很尖、很集中，那这个预测就应该更可信。
- 点被遮挡了，或者图像里信息很模糊，heatmap 会比较散，甚至出现多个峰，那这个预测就应该少信一点。

所以我们希望给每个点再配一个 **confidence（置信度）**，用来表示“这个 heatmap 预测得靠不靠谱”。

这个置信度后面主要有两个用途：

1. 给多视角三角化加权
2. 给多视角特征融合加权

也就是说，置信度高的视角，影响更大；置信度低的视角，影响更小。

---

## 2. 我们不想要什么样的置信度？

有两种很常见但不太理想的做法：

### 2.1 直接取 heatmap 的最大值

最简单的方法就是：

```text
confidence = max(H)
```

也就是谁的峰值高，谁就更自信。

这个方法的问题是：

- 它只看最高那个点，不看整张热图的形状。
- 有时候虽然峰值高，但整张热图其实很乱，或者有多个峰，这种情况也不应该算高置信度。
- 很容易整体都偏高，动不动就是 0.9 以上，不利于区分样本质量。

### 2.2 用 1 / std 这种无界形式

另一种思路是：heatmap 越集中，方差越小，所以可以用：

```text
confidence = 1 / std
```

这个思路方向是对的，但问题是：

- 数值范围不稳定，不是 0 到 1。
- 容易出现尺度很奇怪的问题，不方便可视化，也不方便直接解释。
- 它虽然反映“集中不集中”，但还是没有显式考虑“是不是多峰、是不是很混乱”。

---

## 3. 我们现在的思路：同时看“集中度 + 峰值 + 熵”

我们现在的置信度不是只看某一个指标，而是把三个信息综合起来：

1. **方向上的集中程度**
2. **峰值是否明显**
3. **整张 heatmap 是否混乱**

这样做的直觉很简单：

- 如果一个点预测得准，那么热图应该既集中、又有明显主峰、而且整体比较干净。
- 如果这个点被遮挡或者不确定，那么热图一般会更散、更平、更乱，置信度自然就会下降。

---

## 4. 先把 heatmap 看成概率分布

假设某个点的 heatmap 记作：

```text
H(u, v)
```

我们先把它归一化，让它满足：

```text
sum_u sum_v H(u, v) = 1
```

这样它就可以看成一张二维概率分布图。

---

## 5. 第一部分：看 heatmap 在 x 和 y 方向上散不散

我们先分别对 x 和 y 方向做边缘分布：

```text
p_u(u) = sum_v H(u, v)
p_v(v) = sum_u H(u, v)
```

然后计算它们的均值和方差。

```text
mu_u = sum_u p_u(u) x_u
mu_v = sum_v p_v(v) y_v

sigma_u^2 = sum_u p_u(u) (x_u - mu_u)^2
sigma_v^2 = sum_v p_v(v) (y_v - mu_v)^2
```

这里：

- `sigma_u` 越小，说明 heatmap 在横向越集中
- `sigma_v` 越小，说明 heatmap 在纵向越集中

为了把它变成一个 0 到 1 之间的分数，我们再做一次归一化：

```text
s_u = 1 - clip(sigma_u / sigma_uni, 0, 1)
s_v = 1 - clip(sigma_v / sigma_uni, 0, 1)
```

这里的 `sigma_uni = 1 / sqrt(12)`，对应的是“完全均匀分布”在 [0,1] 上的标准差。

这样一来：

- 如果 heatmap 很集中，`s_u` 和 `s_v` 就接近 1
- 如果 heatmap 很散，`s_u` 和 `s_v` 就接近 0

这部分本质上是在回答：

> 这个热图在空间上到底集中不集中？

---

## 6. 第二部分：看主峰够不够明显

接着我们看热图里的最高点：

```text
p_max = max(H)
```

如果主峰很明显，通常说明模型比较确信这个位置。

但我们不直接用 `p_max`，因为直接用的话容易过于敏感，也容易很多点都偏高。
所以我们做了一个对数压缩：

```text
r_peak = max(N * p_max, 1)
s_peak = clip(log(r_peak) / log(N), 0, 1)
```

其中 `N = h * w`，表示 heatmap 总共有多少个格子。

这一步的作用是：

- 峰值越高，`s_peak` 越大
- 但增长不会太激进，不会轻易全冲到 0.9 以上

这部分回答的是：

> heatmap 有没有一个足够明显的主峰？

---

## 7. 第三部分：看整张 heatmap 乱不乱

有时候即使主峰存在，整张图还是可能很乱，比如：

- 附近还有很多次峰
- 多个区域同时亮起来
- 热图整体比较“脏”

这种时候，只看峰值是不够的，所以我们又加了一个 **熵（entropy）**：

```text
E = - sum_u sum_v H(u, v) log(H(u, v) + epsilon)
```

然后把它归一化成一个“越小越乱，越大越干净”的分数：

```text
s_ent = clip(1 - E / log(N), 0, 1)
```

这样：

- 如果 heatmap 非常集中、结构很清楚，熵就小，`s_ent` 就高
- 如果 heatmap 很混乱、很分散、多峰，熵就大，`s_ent` 就低

这部分回答的是：

> 整张 heatmap 的结构是不是清晰、单一、干净？

---

## 8. 先得到一个全局质量分数

我们把“峰值明显程度”和“整体干净程度”平均一下：

```text
s_g = 0.5 * (s_peak + s_ent)
```

所以 `s_g` 可以理解成：

> 这张 heatmap 从整体上看，质量怎么样？

---

## 9. 最后把这些信息合起来，得到置信度

最终的 axis-wise confidence 定义为：

```text
c = ( sqrt([s_u, s_v] * s_g) )^gamma
```

其中：

- `[s_u, s_v]` 表示 x 和 y 两个方向各自的集中度分数
- `s_g` 是全局质量分数
- `gamma = 1.2`

最后再把它裁剪到 `[0, 1]`。

也就是说：

- 如果热图在 x/y 方向都很集中，而且整体又有明显主峰、低熵，那 confidence 就高
- 如果热图很散、很乱、多峰，那 confidence 就低

如果后面某个模块只需要一个标量 confidence，也可以简单取平均：

```text
c_bar = (c_u + c_v) / 2
```

---

## 10. 这样设计的好处是什么？

### 10.1 更符合真实逻辑

这个 confidence 不是随便拍脑袋出来的，而是和“这个 heatmap 到底靠不靠谱”直接相关。

直觉上：

- 预测越准，热图越集中、越干净、越单峰，confidence 越高
- 遮挡越严重、歧义越大，热图越散、越乱，confidence 越低

### 10.2 能反映不确定性，而不是只看峰值

它不会像 `max(H)` 那样，只看最高点。
它会同时考虑：

- 分布是不是集中
- 主峰是不是突出
- 整体是不是混乱

所以更全面。

### 10.3 数值范围稳定

我们把最终 confidence 压到 `[0, 1]` 之间，这样：

- 好解释
- 好可视化
- 好用于后续加权
- 不容易出现数值爆炸

### 10.4 不容易一股脑全顶到高分

我们专门用了：

- `log` 压缩 peak
- `gamma` 压缩最终输出

目的就是避免“明明只是普通质量，也都冲到 0.9+”这种情况。

---

## 11. 它和遮挡的关系是什么？

这个方法不是显式预测“有没有遮挡”的二分类结果，
但它会通过热图形状间接反映遮挡。

因为真实情况下，一旦某个点被遮挡，模型通常会出现这些现象：

- 主峰变弱
- 热图变散
- 多个位置都不太确定
- 熵变大

而我们的方法正好就是在测这些东西。

所以虽然没有单独监督一个 occlusion label，
但它天然具备一定的遮挡敏感性。

---

## 12. 这个 confidence 目前在系统里怎么用？

当前主要用在两个地方：

### 12.1 多视角三角化加权

不同相机视角对同一个点的预测质量不一样。
有了 confidence 之后，就可以：

- 高置信度视角多给权重
- 低置信度视角少给权重

这样三角化会更稳。

### 12.2 多视角特征融合加权

后续做 view fusion 时，也可以根据 confidence 决定：

- 哪些视角更值得信
- 哪些视角只是辅助

这样能减少坏视角对最终结果的干扰。

---

## 13. 一句话总结

如果用一句话概括这套方法，就是：

> 我们不是把 confidence 当成一个额外拍脑袋的分数，而是直接从 heatmap 的形状里去衡量“这个预测有多集中、多清楚、多确定”。

所以它本质上是一个 **基于热图分布形状的质量估计分数**。

---

## 14. 最简公式版

如果只保留最核心的形式，可以写成：

```text
p_u(u) = sum_v H(u,v)
p_v(v) = sum_u H(u,v)

s_u = 1 - clip(sigma_u / sigma_uni, 0, 1)
s_v = 1 - clip(sigma_v / sigma_uni, 0, 1)

s_peak = clip(log(max(N * max(H), 1)) / log(N), 0, 1)
s_ent = clip(1 - E / log(N), 0, 1)

s_g = 0.5 * (s_peak + s_ent)

c = ( sqrt([s_u, s_v] * s_g) )^gamma
```

其中：

- `sigma_u, sigma_v` 表示两个方向的扩散程度
- `s_peak` 表示主峰显著性
- `s_ent` 表示整体混乱程度
- `s_g` 表示全局质量
- `c` 就是最终 confidence

---

## 15. 后面如果还想继续升级，可以怎么做？

这套方法已经比单纯 peak 或 1/std 合理很多了，但如果以后想再往前走，还有两个方向：

### 15.1 和真实误差做 calibration

比如把 confidence 和 reprojection error、triangulation residual 挂钩，
让它不仅“看起来合理”，而且和真实误差更一致。

### 15.2 单独学一个 uncertainty 分支

也可以在 heatmap 之外，再学一个专门的 uncertainty head。
不过那样参数更多，训练也更复杂。

目前这版的优点是：

- 不额外加太多结构
- 解释性强
- 比较容易落地
- 对多视角系统很友好
